# Session Review: branch policy v1

Date: 2026-05-15 00:10
Scope: production branch としての `main` 運用ルール整理
Summary: Cloudflare Pages の production deploy が `main` に直結する前提を踏まえ、`main` 直作業を避ける branch policy を `AGENTS.md` と `.codex/rules/change-policy.md` に整理した。今後の通常作業は `codex/...` branch を前提とする。

## 1. セッションの目的

- `main` が production branch である前提をルール化する
- `main` 直作業を避ける branch 運用を固定する
- `AGENTS.md` と rule file の役割分担を明確にする

## 2. 実際にやったこと

- `AGENTS.md` には導入サマリーだけを短く追記した
- `.codex/rules/change-policy.md` に branch 運用ルールを追記した
  - `main` は production branch
  - `main` で直接作業しない
  - 通常作業は `main` から `codex/...` branch を切る
  - `main` 反映前は最小変更と build / 出力確認を優先する
  - `data/` は原則 collector workflow 由来

## 3. 変更したもの

### Files

- `AGENTS.md`
- `.codex/rules/change-policy.md`

## 4. 判断・意思決定

- 運用ルールは `docs/` ではなく rule file で管理する方針を採用
- `AGENTS.md` は入口に留め、詳細は `.codex/rules/change-policy.md` に寄せる
- 以後の通常作業は branch 前提とし、`main` 直作業は例外扱いとする
